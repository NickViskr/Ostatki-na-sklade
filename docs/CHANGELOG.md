- Оптимизирована функция `saveExternalShipments` in `Code.gs`: внедрено поколоночное сравнение (`setIfChanged`) значений перед записью, предотвращая лишнюю перезапись одинаковых данных в ячейках Google Таблиц и исключая ложные инкременты `updatedCount`.
- Добавлена строгая проверка непустых значений для даты отгрузки (`shipmentDate`) и состава (`itemsJSON`) при статусе `new`, защищая от случайного стирания ранее полученных данных пустыми значениями от прокси-сервера.

### Изменено
- Логика проведения транзакций (`commitTransaction`): для типа 'virtual' при Расходе себестоимость главной строки сбрасывается в 0, а собственный складской остаток артикула комплекта не изменяется (списываются только компоненты).
- Логика удаления транзакций (`deleteTransaction`): при откате главной строки Расхода виртуального комплекта (с нулевой себестоимостью и заполненным `groupId`), собственный остаток артикула не возвращается на склад, возвращаются только компоненты.
- Валидация остатков перед отгрузкой: для `legacy` проверяется собственный остаток комплекта и компонентов, для `virtual` — только компонентов.
- Исправлена формула расчета `nCap` при каскадном удалении транзакции комплекта: теперь прибавляется прямая сумма себестоимости списания из соответствующей колонки вместо умножения количества на цену за единицу, так как в `Себестоимость списания` теперь пишется итоговая себестоимость (сумма) списания.
