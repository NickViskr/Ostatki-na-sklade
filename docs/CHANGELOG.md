При этом в строку транзакции корректно пишется вычисленный `writeOffCost` (средняя себестоимость до списания × списанное количество) для сохранения аналитических отчётов.
- Оптимизировано восстановление удаленных транзакций в `restoreMultipleArchivedItems` (`Code.gs`): вместо деструктивного полного пересчёта склада `recalculateStockFully` реализован инкрементальный откат остатков на базе накапливания изменений в `stockChanges` (с обратными знаками: +qty для 'Приход', -qty для 'Расход'). Добавлена проверка виртуальных комплектов, предотвращая ложное изменение остатков. В случае ухода остатка в минус, значение принудительно устанавливается в 0 с выводом предупреждения.
