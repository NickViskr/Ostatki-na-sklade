# Архитектура проекта

## Общая структура
Проект представляет собой full-stack приложение, построенное на интеграции Google Apps Script и современного React-интерфейса:

1. **`server.ts`** — Прокси-сервер, обрабатывающий локальные запросы разработчика и интеграцию с внешними сервисами (включая Gemini API).
2. **`Code.gs`** — Серверная бизнес-логика, выполняющаяся в среде Google Apps Script. Непосредственно взаимодействует со встроенной СУБД на базе Google Таблиц.
3. **`src/`** — Клиентская часть приложения (React, TypeScript, Zustand, Tailwind CSS, Lucide).

---

## Назначение листов Google Таблицы

- **SKU** — Реестр уникальных товарных позиций. Содержит SKU, ШТ/КОР, минимальный остаток и штрихкоды.
- **Остатки** — Актуальные остатки товаров на складе, расчет средней себестоимости по методу средневзвешенной цены и капитализации остатков.
- **Транзакции** — Лог всех движений товаров (Приход, Расход, Списание, Инвентаризация, Услуги).
- **Комплекты** — Спецификации комплектов. Включает связку комплекта с компонентами, норму расхода (`quantity`) и тип комплекта (`kitType`: `legacy` / `virtual`).
- **Справочники** — Вспомогательные данные для выпадающих списков (Объекты, Пользователи и др.).

## Схема потока данных при Расходе (отгрузке)

```
[Пользователь нажимает "Отгрузить"]
               │
               ▼
   [Валидация остатков (клиент/сервер)] ──► Если виртуальный: проверяет только компоненты
               │                            Если legacy: проверяет компоненты + сам артикул
               ▼
 [Запись транзакции (commitTransaction)]
               │
               ├─► [Запись главной строки] ──► Если виртуальный: writeOffCost = 0, не меняет остаток комплекта
               │                              Если legacy: списывает остаток комплекта по средней цене
               │
               └─► [Запись строк компонентов] ──► Списывает каждый компонент по его средней цене,
                                                  связывая строки через kitGroupId
               │
               ▼
 [Обновление листа "Остатки" и синхронизация]
```
